PRE-INSTALL PREPARATION

Setup Windows Server sebelum install komponen apapun

PRE-REQ ▼
KomponenMinimumRecommended

CPU2 Core4 Core

Disk D: (PI Data)20 GB50 GB+

01
Set Hostname Final (SEBELUM minta license!)
# PowerShell (Run as Administrator) Rename-Computer -NewName "PISERVER01" -Restart
02
Set Static IP Address
New-NetIPAddress -InterfaceAlias "Ethernet" \ -IPAddress 192.168.1.100 \ -PrefixLength 24 \ -DefaultGateway 192.168.1.1 Set-DnsClientServerAddress -InterfaceAlias "Ethernet" \ -ServerAddresses 127.0.0.1
03
Install IIS + Fitur Windows yang Dibutuhkan
Install-WindowsFeature -Name \ Web-Server, Web-Asp-Net45, Web-Windows-Auth, \ Web-Net-Ext45, Web-ISAPI-Ext, Web-ISAPI-Filter, \ Web-Static-Content, Web-Default-Doc, \ Web-Http-Errors, Web-Http-Logging, \ Web-Request-Monitor, Web-Filtering, \ Web-Mgmt-Console \ -IncludeManagementTools # .NET Framework 3.5 (dibutuhkan PI AF) Install-WindowsFeature Net-Framework-Core
04
Disable IE Enhanced Security & Setup Firewall
# Buka port PI Server (TCP 5450) New-NetFirewallRule -DisplayName "PI Server" \ -Direction Inbound -Protocol TCP \ -LocalPort 5450 -Action Allow # Buka port HTTP/HTTPS untuk PI Vision New-NetFirewallRule -DisplayName "PI Vision HTTP" \ -Direction Inbound -Protocol TCP \ -LocalPort 80,443 -Action Allow

KONFIGURASI PI DATA ARCHIVE

Setup security, PI Points/Tags, dan verifikasi data store

CONFIG ▼
01
Setup PI Identity & Security Mapping (PI SMT)
PI SMT → Security → Identities, Users & Groups New Identity: "PI_Readers" ← untuk user read-only New Identity: "PI_Writers" ← untuk interface/data input PI SMT → Security → Mappings & Trusts → Mappings Windows User: PISERVER01\Administrator PI Identity: piadmins → Save
02
Buat PI Points (Tags) untuk Testing
PI SMT → Data → Point Builder → New Point --- Contoh Tag 1 --- Point Name: Plant.Temp.Reactor01 Point Type: Float32 Eng Units: DegC Span: 0 – 200 Compression: ON, DevPercent: 0.5 --- Buat juga tag berikut --- Plant.Pressure.Vessel01 Float32 barg 0-10 Plant.Flow.Feed01 Float32 m3/h 0-100 Plant.Level.Tank01 Float32 % 0-100 Plant.Status.Pump01 Int32 — 0-1
03
Test Input Data Manual
PI SMT → Data → Archive Editor Tag: Plant.Temp.Reactor01 Timestamp: * (sekarang) Value: 85.5 → Enter → Verifikasi nilai masuk
04

SETUP PI ASSET FRAMEWORK (AF)

Buat hierarki aset, database AF, template, dan attribute mapping

CONFIG ▼
01
Buat AF Database Baru (PI System Explorer)
# Buka PI System Explorer Start Menu → PI System Explorer Connect to AF Server: PISERVER01 File → New Database Name: PlantDatabase Description: Demo Plant Asset Model → OK

Bangun Hierarki Aset (Element Tree)
# Struktur hierarki di panel kiri PlantDatabase └─ Plant Site A ├─ Utility Area │ ├─ Reactor 01 │ └─ Storage Tank 01 └─ Feed Area └─ Feed Pump 01 # Klik kanan pada node → New Element
03
Buat Element Template
Library → Element Templates → New Template Name: ReactorTemplate # Tambah Attributes (mapping ke PI Tags): Attribute: Temperature Type: PI Point PI Tag: Plant.Temp.Reactor01 Unit: DegC Attribute: Pressure Type: PI Point PI Tag: Plant.Pressure.Vessel01 Unit: barg Attribute: OperatingStatus Type: PI Point PI Tag: Plant.Status.Pump01
04
Apply Template & Check-In
# Klik "Reactor 01" → Properties Template: ReactorTemplate → Apply Template → Check-in (Ctrl+K) # Verifikasi di tab Attributes: # Temperature harus tampil nilai dari PI tag ✅

INSTALL PI VISION

Web-based visualization layer di atas IIS

INSTALL ▼
01
Verifikasi IIS Berjalan
Get-Service W3SVC # Jika Stopped: Start-Service W3SVC # Test di browser: http://localhost # Harus tampil IIS default page ✅

Jalankan PI Vision Installer
# Extract dan jalankan: PI-Vision_2022_.exe → setup.exe (Run as Admin) --- Wizard Step 1: Prerequisites --- # Semua harus hijau ✅, jika ada merah: fix dulu! --- Wizard Step 2: IIS Configuration --- Website: Default Web Site Virtual Dir: PIVision App Pool: PIVisionServiceAppPool --- Wizard Step 3: AF Server --- AF Server: PISERVER01 Database: PlantDatabase → Test Connection → Connected ✅ --- Wizard Step 4: Service Account --- Account: Network Service (untuk lab/dev)

Konfigurasi Post-Install — IIS
IIS Manager → Application Pools → PIVisionServiceAppPool → Advanced Settings Identity: Network Service Load User Profile: True IIS Manager → Sites → Default Web Site → PIVision → Authentication Windows Authentication: ENABLED ✅ Anonymous Authentication: DISABLED ❌

Setup PI Security untuk PI Vision
--- PI SMT: Trust untuk PI Vision --- Security → Mappings & Trusts → Trusts → New Trust Name: PIVisionTrust NetMask: 127.0.0.1 (localhost) PI Identity: PI_Readers → Save --- PI System Explorer: AF Security --- File → Server Properties → Security Add: PISERVER01\NetworkService Role: PI AF Reader

BUAT DISPLAY PI VISION

Trend, value, gauge, multi-state — setup tampilan real-time

Save Display
File → Save Display Name: Reactor01_Overview Folder: /PlantDisplays/ → Save → Klik "Design" → toggle ke "View Mode"

KONEKSI DATA SOURCE

OPC DA, Modbus, atau PI Random untuk simulasi data

INTERFACE ▼
A
Option A: PI Interface untuk OPC DA
# Install PI Interface for OPC DA PI ICU → Add Interface → OPC DA Host: PISERVER01 Interface ID: 1 OPC Server: localhost\OPCSimulator Scan Class: 10 detik → Build PI Tags dari OPC Items → Start Interface
B
Option B: PI Interface untuk Modbus TCP
PI ICU → Add Interface → Modbus TCP Device IP: 192.168.1.200 (IP PLC/RTU) Port: 502 Scan: 5 detik # Mapping register ke PI Tag: HR 40001 → Plant.Temp.Reactor01
C
Option C: PI Random Interface (Simulasi — untuk Lab)
# Tidak perlu hardware! Langsung simulasi nilai random PI ICU → Add Interface → PI Random → Assign ke semua PI Tags yang sudah dibuat → Scan Class: 5 detik → Start Interface # Nilai akan berubah random sesuai span tag # Langsung bisa dilihat di PI Vision ✅
D
Option D: PI Performance Monitor (CPU/RAM → PI Tag)
# Push Windows performance counters sebagai PI tag PI ICU → Add Interface → PI Performance Monitor → Auto-discover Windows counters # Contoh mapping: \Processor(_Total)\% Processor Time → Server.CPU.Usage \Memory\Available MBytes → Server.RAM.Available
